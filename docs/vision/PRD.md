# 自分コンテキスト (Personal Context Engine) - 製品要件定義書 (PRD)

## 1. プロダクトのビジョン

特定の LLM エコシステムから独立した、ポータブルな個人の人生コンテキストデータベースを構築する。

**脱・LLM ロックイン**を実現し、ユーザーは AI 主導の**受け身型インタビュー**に答えるだけで、自己の構造化された知識基地（ファクトと人生年表）を自動構築し、いつでも任意の LLM へプレーンな Markdown 形式で持ち出せる世界を目指す。

## 2. ターゲットユーザー

- **フェーズ1（OSS版）**: セルフホストやローカル環境での実行が可能なエンジニア層。
- **フェーズ2・3（Web版・コンサルティング）**: 環境構築の手間を嫌うリテラシーの低い層、および自身のコンテキストを言語化することに課題を感じている中小企業の経営者・シニア層。

## 3. ユーザー体験（UX）の原則

- **受け身型入力の徹底**: 多忙な現代人がゼロから自己履歴を記述する心理的・肉体的ハードルを排除する。システムからの自律的な質問に答えるだけの**受け身型**インタラクションに限定する。

- **継続性の担保**: ユーザーの負担を最小化するため、1日の質問数を制限し、離脱インテント（疲労や飽きのサイン）を検知した場合は AI が自律的にセッションを終了する。

## 4. 機能要件 (Must / Should / Want)

- **Must（必須要件）**
  - SQLite を唯一の Source of Truth としたデータ永続化と履歴管理。
  - システムが保持するベース質問（DB 初期データとして投入）と、LLM による自然なコーチングトーンへの変換機能。
  - ユーザーの回答に対する LLM の深掘り（最大2回）。
  - 会話からのファクトおよび人生年表の自動抽出と構造化保存（追記型：競合時も新規レコード追加）。
  - LLM（Claude / ChatGPT 等）へインポート可能な Markdown 形式でのエクスポート機能。
  - データの公開範囲（`public` / `private`）制御。
  - **音声入力**: ブラウザからの録音 → ローカル Whisper によるテキスト変換 → チャットフローへの合流。
  - マルチ LLM 対応（Claude API / Ollama 等を `LLMProvider` インターフェースで抽象化）。

- **Should（推奨要件）**
  - ユーザーの離脱インテントおよび文字数減少の自動検知とセッション終了機能。
  - ローカル実行と Web 版（マルチテナント）の双方に対応可能な、`user_id` をベースとしたスキーマ設計。
  - データの暗号化・復号化インターフェースの抽象化（`EncryptionProvider`）。

- **Want（希望要件）**
  - 外部ソース（Markdown ファイル等）からのインポート機能。
  - ファクトの重複検知・統合機能。

## 5. 非機能要件

- **データのポータビリティ**: データはプラットフォームに依存せず、常に Markdown 等のプレーンテキストで出力可能であること。
- **将来のマルチテナント化への配慮**: 全てのデータテーブルにおいて `user_id` を必須とし、スムーズな SaaS 化（フェーズ2）を担保する。
- **セキュリティとデータ保護**:
  - データベースへの保存前に機密データを暗号化・復号化する抽象化レイヤーを設け、Web 版への移行時に環境変数の切り替えのみで堅牢なセキュリティを実現する。
  - **サプライチェーン攻撃対策**: CI/CD では `pnpm install --frozen-lockfile` を必須とし、`pnpm audit` を自動実行する。Socket.dev による依存パッケージの静的解析を導入し、Renovate Bot で依存関係を定期的に更新する。
