# Personal Context Layers — 個人コンテキスト層モデル

## Why This Model Exists

AIに「自分らしい」回答をさせるには、AIが自分のことを深く理解している必要がある。  
しかしその「自分」は一枚岩ではない。

変わらない信念もあれば、経験によって形成された性格もあり、場面によって変わる現在地もある。  
この違いを無視してすべてを一括でシステムプロンプトに投げ込むのは非効率で、かつ不自然である。

このモデルは**「どの情報をどのタイミングでAIに渡すか」を構造化する**ために設計した。

## 3ゾーン同心円モデル

```
╔══════════════════════════════════════════╗
║                 S T A T E                ║
║   ┌──────────────────────────────────┐   ║
║   │           S H A P E              │   ║
║   │   ┌──────────────────────────┐   │   ║
║   │   │        C O R E           │   │   ║
║   │   │    価値観・信念・気質       │   │   ║
║   │   └──────────────────────────┘   │   ║
║   │  経歴・経験・癖・スタンス・関係性      │   ║
║   └──────────────────────────────────┘   ║
║       今の目標・プロジェクト・好み            ║
╚══════════════════════════════════════════╝
```

**安定性の軸**：内側ほど「変わらないもの」、外側ほど「状況で変わるもの」

## 3ゾーン定義

### CORE（核）— 変わらない自己

その人を根底で規定するもの。人生を通じてほぼ変化しない。  
いわば「自分自身のシステムプロンプト」。

| Layer | 名称                                 | 説明                                                                       |
| ----- | ------------------------------------ | -------------------------------------------------------------------------- |
| L1    | **Values & Beliefs（価値観・信念）** | 何を大切にするか。曲げられない軸。                                         |
| L2    | **Character（気質・性格）**          | 生まれ持った傾向。内向的/外向的、直感型/論理型、繰り返すパターンの根っこ。 |

### SHAPE（形）— 経験が形作ったもの

人生の出来事・学習・関係性によって蓄積されたもの。変化するが緩やか。

| Layer | 名称                                        | 説明                                                                                               |
| ----- | ------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| L3    | **Life Timeline（人生年表）**               | 幼少期から現在までのざっくりした年表。転換点・転居・転職・病気など人生の節目。                     |
| L4    | **Professional Profile（職務詳細）**        | 各仕事・プロジェクトで自分が何をしてきたか。スキル・役割・成果・失敗の詳細記録。                   |
| L5    | **Relationships（関係性）**                 | 家族・友人・メンター・コミュニティ。合う人・合わない人のパターンも含む。                           |
| L6    | **Opinions & Stance（意見・スタンス）**     | 技術・社会・ビジネス・働き方に対する具体的な立場。「これは過大評価」「これは間違っている」など。   |
| L7    | **Fears & Avoidance（恐れ・回避パターン）** | 何が怖いか、何から逃げがちか。人間の行動の多くは「こうなりたくない」で動いている。                 |
| L8    | **Recurring Patterns（繰り返す癖）**        | 仕事や人間関係で繰り返してきたパターン。追い込まれると連絡を絶つ、完璧主義で完成させられないなど。 |

### STATE（現在地）— 今この瞬間の文脈

内側のCORE/SHAPEが今の状況に応じて表れる形。状況・目的・気分で変動する。

| Layer | 名称                                      | 説明                                                                                                               |
| ----- | ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| L9    | **Goals & Direction（目標・方向感）**     | 今何を達成しようとしているか。今期のプロジェクト、直近の悩み、今どこへ向かっているか・何から離れようとしているか。 |
| L10   | **Preferences & Style（好み・スタイル）** | 文体、作業スタイル、好きなもの。AIとのやり取りのトーンに直結する。                                                 |

## LLMへの渡し方

| ゾーン    | タイミング                     | 用途                                                         |
| --------- | ------------------------------ | ------------------------------------------------------------ |
| **CORE**  | 常にシステムプロンプトに含める | すべての会話の土台。これがないとAIは「他人」として振る舞う。 |
| **SHAPE** | 文脈に応じて選択的に追加       | 転職相談にはL4、人間関係の悩みにはL5・L7・L8を追加するなど。 |
| **STATE** | タスクごとに注入               | 「今日の目標」「このプロジェクトの背景」など都度渡す。       |

---

## なぜ同心円か

**レイヤー積み上げ図**だと「上下関係（重要度の差）」に見えてしまう。  
**同心円**だと「包含関係（内側が外側を規定する）」が直感的に伝わる。

核（CORE）があるから形（SHAPE）が意味を持ち、形があるから現在地（STATE）が一貫性を持つ。  
これはコンポーネントの依存関係ではなく、**存在の構造**だ。

---

## Context Engineering との接続

2025年に「Context Engineering（文脈工学）」という概念が注目された。  
AIシステムに何を・いつ・どう渡すかを設計する分野だ。

これまでのContext Engineeringは**AIシステム側**（RAG、ツール、メモリ設計）に集中していた。  
**人間側のコンテキスト構造**は未開拓のままだ。

このモデルはその空白を埋める試みである。

> _「AIに自分を理解させる」のではなく、「自分を構造化してAIに渡せる形にする」_

---

## このプロジェクトの位置づけ

personal_context は、このCORE/SHAPE/STATEモデルを実装するツールだ。  
システム的な機能（インタビュー・DB・エクスポート）は手段に過ぎない。

**ゴールは、10のレイヤーを埋め、いつでもどのAIにも渡せる「自分のコンテキスト」を持つこと。**
