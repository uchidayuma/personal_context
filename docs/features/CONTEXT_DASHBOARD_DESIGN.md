# Context Dashboard 設計書

## 問題

現状のUIはチャット画面だけで、以下が分からない：
- 自分のコンテキストに何が入っているか
- どのレイヤーがどれだけ埋まっているか
- あとどれくらいインタビューすれば「使える」状態になるか

インタビューを続けるモチベーションがわかない。

## 解決策：コンテキストダッシュボード

新しいビュー「マイコンテキスト」を追加する。  
スタンプカード形式でレイヤーごとの充足度を可視化し、  
実際に収集されたデータもその場で確認できるようにする。

## レイヤー充足度の計算ロジック

各レイヤーは「何件のデータがあるか」で充足度を算出する。  
完全自動 — ユーザーが何かを入力する必要はない。

| Layer | データソース | 充足しきい値（100%） |
|-------|-------------|---------------------|
| L1 価値観・信念 | `structured_facts` category: values, beliefs | 5件 |
| L2 気質・性格 | `structured_facts` category: character, personality | 3件 |
| L3 人生年表 | `life_timeline` 件数 | 10件 |
| L4 職務詳細 | `professional_records` 件数 | 3件 |
| L5 関係性 | `structured_facts` category: relationships | 5件 |
| L6 意見・スタンス | `structured_facts` category: opinions, stance | 5件 |
| L7 恐れ・回避 | `structured_facts` category: fears, avoidance | 3件 |
| L8 繰り返す癖 | `structured_facts` category: patterns, habits | 3件 |
| L9 目標・方向感 | `structured_facts` category: goals + `life_timeline` 現在以降 | 3件 |
| L10 好み・スタイル | `structured_facts` category: preferences, style | 3件 |

充足度 = min(実件数 / しきい値, 1.0) × 100%

## 新規APIエンドポイント

```
GET /api/progress

Response:
{
  overall: number,          // 全体充足度 (0-100)
  layers: [
    {
      id: "L1",
      name: "価値観・信念",
      zone: "CORE",
      percent: number,      // 0-100
      count: number,        // 実件数
      threshold: number     // しきい値
    },
    ...
  ],
  totals: {
    facts: number,
    timeline: number,
    professional: number,
    vignettes: number
  }
}
```
