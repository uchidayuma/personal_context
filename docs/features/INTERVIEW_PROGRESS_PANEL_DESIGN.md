# インタビュー中プログレスパネル 設計書

## 問題

インタビュー（Chat画面）中は `/api/progress` の情報が一切見えない。  
「どれだけ溜まったか」が分からないため、インタビューを続けるモチベーションが持続しない。

既存の「マイコンテキスト」ダッシュボードはページを移動しないと見られず、インタビューの流れを断ち切る。

## 解決策：チャット内プログレスヘッダー

Chat画面の上部に常時表示される折りたたみ式のミニプログレスパネルを追加する。

---

## 更新タイミング

- コンポーネントマウント時（セッション開始時）に1回 fetch
- AI の応答が返ってくるたびに再 fetch（`sendMessage` 完了後）
- 音声モードからチャットに戻ったときも再 fetch

---

## 技術仕様

### コンポーネント構成

```
Chat.tsx
  └─ ProgressHeader.tsx  (新規)
       ├─ 全体進捗バー
       ├─ L1〜L10 ドット行
       └─ 展開時レイヤー詳細
```

### データ取得

既存の `/api/progress` エンドポイントを使用。新しいAPIは不要。

```typescript
// Chat.tsx に追加
const [progress, setProgress] = useState<ProgressData | null>(null)

async function refreshProgress() {
  const res = await fetch('/api/progress')
  if (res.ok) setProgress(await res.json())
}

// sendMessage() 完了後に呼び出す
useEffect(() => { refreshProgress() }, [])
```

### ドット表示ロジック

```typescript
// threshold到達 → ◉ (緑、達成色)
// count > 0     → ◎ (黄、進行中)
// count === 0   → ○ (グレー、未着手)
```

### レイアウト

Chat.tsx の `.container` を変更せず、`.container` の上部（`messages` の前）にパネルを配置。  
パネル展開時は `.messages` エリアが縮小（flex レイアウトで自然に対応）。

---

## デザイン原則

- **常時表示**: 折りたたみ時でも全体 % とドット行は常に見える → 意識的に確認しなくても進捗が目に入る
- **邪魔しない**: デフォルト折りたたみ、高さ 56px 以内 → チャットの流れを妨げない
- **リアルタイム更新**: AI 応答のたびに自動更新 → 回答するたびに進捗が増える体験
